# Notes Inbox

Telegram captures and untriaged project notes land here first. This file is an inbox, not a priority list.

## Triage workflow

1. Capture an idea, bug, URL, observation, or rough post idea here.
2. Codex performs the review when asked for a triage pass, or when a task requires a related note. Codex deduplicates the note and adds context: affected page, type, impact, and the next action.
3. Move actionable work to `TODO_content.md` or `TODO_design.md`.
4. Put only the next few committed outcomes in `TODO.md`.
5. Mark or remove the inbox entry only after it has been placed deliberately.

## Ownership

The Telegram bot captures notes. Agents own their organization and movement between the Markdown files. Capture does not automatically create a priority or a commitment to implement the note.
