# Git Workflows and Commands

When interacting with this repository via Git, follow these strict rules:

1. **NEVER Push Without Explicit Permission**: Under no circumstances should you run `git push` without asking the user for explicit permission first.
2. **Commit Only When Necessary**: When creating commits, provide a clear, concise commit message.
3. **Ask Before Committing Extensive Changes**: If the changes are large or affect many files, confirm with the user before running `git commit`.

If any task seems to imply deploying or publishing, stop at the commit stage (if authorized) and explicitly ask: "Would you like me to push these changes to the repository?"
